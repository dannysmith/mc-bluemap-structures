# Task: Consider using chunkbase data

A little while ago we built the tool in `tools/chunkbase-verify/` - See the readme in there for details. What this does is it uses playwright to actually open chunkbase up for the relevant seed. And given a certain radius, it gets chunkbase's estimates of the locations of these various structures and puts them in `tools/chunkbase-verify/data/` as JSON files.

Now it's occurred to me that We could keep the existing algorithms and systems we have for estimating these things, but introduce some way whereby if a user wanted to, they could run this script and then put that JSON file somewhere appropriate on the server, and they probably have to run the script somewhere where playwright's available, so we might not be able to actually build this in to the mod. But you know, if you imagine saying run that script locally against the seed and then install the mod and put that chunkbase JSON file somewhere, instead of using our algorithms to generate the markers, we could just skip all of that and then just read in that chunk base file, then you use the locations in that. I am obviously conscious it's an enormous file, though. Now I don't think that should be the default.
